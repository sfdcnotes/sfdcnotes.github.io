Lightning Experience 快速入门

发表于 2016-07-08

本文分三个部分：概念，Lightning Experience，Lightning Experience Development。

概念

  • Lightning Experience
    我们看到的全新的 UI 界面

  • Lightning App Builder
    通过拖拽组件生成 Lightning Page

  • Lightning Pages
    Lightning App Builder 拖拽而成的界面，刚接触时很容易混淆，以为是和 Visualforce Pages 类似的东西，但其实不是

  • Lightning Component Framework
    在 Force.com 上的前端框架，可用来开发 Lightning 应用和 Lightning Component

  • AppExchange for Components
    Lightning Component 可以在发布在 AppExchange 上

  • Lightning Design System
    纯 CSS 框架和 Lightning 风格的设计元素

下面的几个曾经名字中带有 Lightning，但和 Lightning Experience 本身没太大关系，现在已经更名

Lightning Process Builder
Lightning Salesforce Connect
Lightning Community Builder
Lightning Schema Builder

Lightning Experience

开启 Lightning Experience

一般来说可以先 demo/sandbox org 试用一下，评估后在 production 开启。
Lightning 可以仅为某些 Profile/Permission set 用户开启。
对已开启的用户，可以配置其登陆 Salesforce 时直接进入 Lightning 还是用户手动切换。

Setups

Setup 从逻辑上更加清新，分为 Administrator，Platform tools，Settings。
所有的配置找不到的可以在 Quick find 上寻找，操作方式和 Classic 没有太大区别。

Navigation Menus

Navigation Menu 类似 Salesforce Classic 的横向导航栏。
Navigation Menu 除了标准的 Item 外，可以自己添加 Tab，包括 Lightning Page Tab 和 Lightning Component Tab。
不同的 Profile 可以分配不同的 Navigation Menu。

其中，
Lightning Page Tab：通过 App Builder 拖拽出 Lightning Page
Lightning Component Tab：自己开发一个 Lightning Component，或者从 AppExchange 下载/购买第三方开发的 Lightning Component

App Builder

App Builder 是一个拖拽工具，将不同的组件拖拽到 Lightning Pages。
Lightning Page 分为 Home Page，Record Page，One Page 三类。
Lightning Page 有固定的模板可以选择，比如一横两栏等等。
组件包括标准组件，Visualforce 页面，开发的 Lightning Component，从 AppExchange 下载/购买的 Lightning Component。
其中，
Home Page 是首页，Record Page 是记录页面，在 App Builder 的感觉类似于 Page Layoout；One Page 即单独的 Lightning Page，可以添加到 Tab 然后加在 Navigation Menu 中。

Limitations

Lightning 相对来讲比较新，虽然每次 Release 都有更新的内容，但是依然有不少限制，比如 Lightning Experience 的 Dashboard 不支持 Visualforce 作为数据源。更多的 Limitation 可以查看下面的相关资料。

Lightning Experience Development

Lightning Experience 和 Salesforce Classic 代码开发不同的地方主要是前端用户交互部分。

Lightning Experience 中的 Lightning Component 可以存在于

Lightning Component Tab 加到 Navigation Menu
嵌入 Lightning Page (One Page/Home Page, Record Page)

值得注意的是，
在 Lightning Experience 中开发的是 Lightning Component，而非 Lightning Application。事实上，Lightning Experience 就是 一个复杂的 Lightning Application。

其中，
加到 Tab 中需要 implements force:appHostable 接口
Lightning Page One Page/Home Page 需要 implements flexipage:availableForAllPageTypes 接口
Lightning Page Record Page 需要 implements flexipage:availableForRecordHome,force:hasRecordId 两个接口
这些在 Developer Console 中新建 Lightning Component 时可以勾选

Lightning Experience 中的 Visualforce 可以存在于

Global Action/Object Action
Visualforce Tab 加到 Navigation Menu
嵌入 Lightning Page

其中，
嵌入 Lightning Page 的 Visualforce 需要将在 Lightning Page 可用打勾

Lightning Experience 中的交互
Lightning Experience 的交互提供了 sforce.one 这个 js 对象和一些 navigation 方法，在写适用于 Lightning Experience 的 Visualforce 需要注意一些交互改为 这种方式。

Visualforce 同时存在于 Lightning Experience 和 Classic
由于 Visualforce 同样存在于 Salesforce Classic，写页面时可以通过 Apex/JavaScript(sforce.one) 判断用户的 Preference 或者当前存在于哪个背景下。这样针对不同的背景显示不同的内容。

Lightning Design System

不管是 Visualforce 还是 Lightning Component 都没有自带 Lightning Experience 的样式。Salesforce 开源了 Lightning Experience 所使用的样式风格。
这是一个纯粹的 CSS 框架，所有 JS 交互需要自己写或者找别人写好现成的。
为了保持和 Lightning Experience 的视觉一致性，可以在 Visualforce 或者 Lightning Component 开发中使用这个 CSS 框架。然而这从某种程度上增加了开发成本，CSS 都要从头写起来。

Lightning Applications

Lightning Application 是一个完整的应用程序。
可以包含多个 Lightning Component，可以暂时认为是写代码版的 App Builder。
Lightning Application 无法存在于 Lightning Experience 中。
事实上，Lightning Experience，Process Builder，Data Import Wizard，App Builder，Package Installation 都是这样一个应用程序。
他们的特点是 url 上 /{namespace}/{appName}.app。
