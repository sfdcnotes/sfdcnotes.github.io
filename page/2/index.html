Salesforce ID中的15位和18位

发表于 2016-03-21

ID 是 Salesforce 记录的主键，在 Salesforce 数据库中，ID 以15位存储，所以在 Salesforce 界面、报表等直接从数据库取数时，ID 显示是15位的。

在 API 中读取数据时 Salesforce 将 ID 从15位转换为18位，所以在Apex、DataLoader、定时导出等外部工具时，ID 显示是18位的。

结论

15位和18位在 Salesforce ID 处理中完全等价。
不管是15位还是18位 ID，都区分大小写。
18位 ID 是 Case Safe 的，换句话说，即便不区分大小写，也不会有两个18位的 ID 出现重复。但如果不区分大小写，两个15位的 ID 有可能出现重复。

双重验证即 Two Factor Authentication，简写为 2FA，又名两步验证、多因素验证，指的是在用户名和密码之外在增加另外的登录凭证，以保证即便密码泄漏，拥有密码的人也无法登录系统。

Trailhead 教你学习 Salesforce

发表于 2016-03-09

Trailhead 是 Salesforce Doc team 推出的一个互动教程，和 Workbook 类似但更具互动性，地址是

https://developer.salesforce.com/trailhead

Trailhead 的名称

Trail 是小径，Trailhead 是小径的起点。Salesforce 中 Trail 就是通向 “Salesforce 技能树” 的小径，Trailhead 是学习 Salesforce 的起点，在 Trailhead 学习 Salesforce 的人都是 Trailblazer（开拓小径的拓荒者）。

Trail, Module, Project

Trailhead 里有许多 Module 和 Project，它们又由一个个 Unit 组成。一些相关的 Module 组成了一个个 Trail，对应着一个个技能，而Project 则相当于 “项目实战”。
